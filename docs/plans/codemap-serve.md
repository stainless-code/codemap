# `codemap serve` — HTTP API (B-tier roadmap item)

> **Status:** in progress · **Issue / backlog:** [`docs/roadmap.md` § Backlog](../roadmap.md#backlog) (`codemap serve` HTTP API). Delete this file when shipped (per [`docs/README.md` Rule 3](../README.md)).

## Goal

Same tool taxonomy + output shape as `codemap mcp`, exposed over HTTP for non-MCP consumers (CI scripts, simple `curl`, IDE plugins that don't speak MCP). Loopback-default; optional `--token` for shared-machine setups.

## Decisions

| #   | Decision                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| --- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| D1  | **HTTP framework.** Bare `node:http` — works in Bun + Node, zero new deps. (Bun.serve is Bun-only; Express/Fastify is overkill for ~12 routes.)                                                                                                                                                                                                                                                                                                        |
| D2  | **Routes.** `POST /tool/{name}` (body = `{<args>}`, response = JSON envelope) for every MCP tool: `query`, `query_batch`, `query_recipe`, `audit`, `context`, `validate`, `show`, `snippet`, `save_baseline`, `list_baselines`, `drop_baseline`. Plus `GET /health` (liveness probe), `GET /tools` (catalog), `GET /resources/{uri-encoded}` mirroring MCP resources (`codemap://recipes`, `codemap://schema`, `codemap://config`, `codemap://index`). |
| D3  | **Binding + auth.** Default bind: `127.0.0.1:7878`. `--host <ip>` to override (refuse `0.0.0.0` unless `--host 0.0.0.0` is explicit). `--port <n>` to change port. `--token <secret>` requires `Authorization: Bearer <secret>` header on every request (skipped on `GET /health` so liveness probes work without leaking the token).                                                                                                                  |
| D4  | **Lifecycle.** Long-running. Bootstrap codemap once at boot (config + resolver — same as MCP). Per request: open DB, run, close DB. SQLite read concurrency handles parallel requests. SIGINT / SIGTERM → graceful shutdown (drain in-flight + close server).                                                                                                                                                                                          |
| D5  | **Output shape.** Match `codemap query --json` envelope (NOT MCP's `{content: [{type: "text", text}]}` wrapper — HTTP doesn't need that transport artifact). Success → 200 + `<envelope>`; error → 4xx/5xx + `{"error": "<msg>"}`. Same shape as `--json` errors.                                                                                                                                                                                      |
| D6  | **Status codes.** 200 success · 400 bad request (invalid JSON, unknown arg, sarif+summary combo) · 401 auth (when `--token` set) · 404 unknown tool · 405 method not allowed · 500 internal.                                                                                                                                                                                                                                                           |
| D7  | **Tool handler reuse.** Tracer 1 extracts `application/tool-handlers.ts` — pure transport-agnostic handlers `(args) => Promise<{ok, payload} \| {ok: false, error}>`. Both MCP and HTTP wrap the same handlers; engine layer untouched.                                                                                                                                                                                                                |
| D8  | **Concurrency.** SQLite supports concurrent readers; each request gets its own `openDb()` / `closeDb()` cycle. `PRAGMA query_only = 1` set per-connection. No queue / pool.                                                                                                                                                                                                                                                                            |
| D9  | **CORS / streaming / WebSockets.** Out of scope v1 — loopback default means browser CORS rarely fires; non-browser tools don't need preflight. Streaming responses deferred (no current consumer).                                                                                                                                                                                                                                                     |
| D10 | **Index freshness.** Server doesn't auto-reindex. Per-request behavior matches MCP: `audit` defaults to running an index prelude (overridable via `no_index: true`); other tools read whatever's in `.codemap.db`. Document: run `codemap` (or `codemap --files <paths>`) externally to refresh; HTTP server picks up the new state on the next request.                                                                                               |
| D11 | **Versioning header.** Every response carries `X-Codemap-Version: <semver>` so consumers can pin / detect upgrades.                                                                                                                                                                                                                                                                                                                                    |

## Out of scope

- **HTTPS / TLS termination.** Loopback default; users wrap with reverse proxy (nginx, Caddy) for TLS.
- **Multi-tenant / per-request `--root`.** Server boots against one project root (same as MCP); multi-root is a v1.x decision when a real consumer asks.
- **WebSocket / SSE / streaming.** No streaming consumer today; deferred.
- **Browser CORS preflight.** Loopback-default makes this unnecessary; documented as a "wrap with proxy" path.
- **Rate limiting.** Loopback only — local CI scripts don't DoS themselves. If `--host 0.0.0.0` ships, revisit.
